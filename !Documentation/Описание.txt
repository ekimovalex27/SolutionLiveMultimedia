Решаемые проблемы:
1. На персональном компьютере пользователя много мультимедиа-файлов, к которым нет доступа из Интернет.
Альтернатива:
Запись части мультимедиа-файлов на мобильное устройство.
2. У пользователей есть мультимедиа-файлы в разных социальных сетях и на сетевых хранилищах.
Чтобы получить доступ к мультимедиа-файлам необходимо заходить в каждую из систем.
Нет унификации интерфейса этих систем, у каждой своя организация доступа к мультмедиа-файлам.
Альтернатива:
Хранение мультимедиа-файлов в одной из систем.
3. Выбор и использование мультимедиа-систем с платным контентом.
Альтернатива:
Например, для изучения иностранного языка купить курс на CD-дисках, сделать копию дисков для дома и автомобиля.

Возможности Live Multimedia System:
1. Пользователю не нужно закачивать файлы на Live Multimedia System для получения доступа к ним.
2. Доступ к мультмедиа с любого компьютерного устройства через веб-браузер.
3. Доступ из приложений, написанных программных платформ (Windows Phone, iOs, Android, Symbian и др.) для телефонов, смартфонов и планшетов.
4. Доступ к мультимедиа на сетевых накопителях.
5. Доступ с автомобильных магнитол (встроенная возможность).
6. Доступ с мобильных устройств (аудио-плееры).
7. Сдача сервиса в аренду сторонним компаниям для предоставления удалённого доступа к их контенту (аудиокниги, аудио и -видео курсы, и т.д.)
8. Использование "толстого" клиента-источника для Windows в режиме запуска службы.
9. Использование "тонкого" клиента-источника (веб-браузер) для любой ОС на компьютере.
10. Доступ к источнику мультмедиа с сервера по ftp! При этом отпадает необходимость использовать клиент-источник на персональном компьютере.
11. Автоматическая конвертация к формату, в котором работает браузер пользователя.
12. Автоматическая адаптация к полосе пропускания пользователя.
13. On-line запись мультимедиа на клиент-приёмник.
14. Радио. Случайное проигрывание чужих песен или по тематической подборке.
15. Доступ к музыкальным магазинам (Apple, Sony и т.д.).
16. Открытие своих радиостанций или аренда существующих для передачи мультимедиа по радио мультплексированного сигнала с индивидуальным
декодированием на каждом зарегистрированном приёмнике (должна быть программная поддержка в приёмнике).
17. Обмен между пользователями мультмедиа-файлами с разных социальных сетей и сетевыми хранилищами без регистрации на них.
18. Создание API OAuth для подключения ресурсов сторонних провайдеров.
20. Создание Playlists, в котором собраны мультмедиа-файлы из разных источников.
21. Сохранение текущей позиции и композиции в список для данной композиции.
22. Сохранение текущего состояния.

Монетизация:
1. Абонентская плата.
Вопросы: зачем платить пользователю, когда он и так получает доступ к своим файлам из социальных сетей и сетевых хранилищ?
Ответ:
- абонентская плата за доступ к своим локальным файлам, доступ к социальным сетям и сетевым хранилищам как бонус. 
- абонентская плата за дополнительные сервисы сторонних провайдеров. Например, музыкальные магазины.
- абонентская плата за дополнительные сервисы. Например, платные системы дистанционного обучения.

2. Бесплатно. Как заработать?
- реклама. Рекламные аудиоролики между песен. Например, если совсем бесплатно, то после каждой песни.
- продажа дополнительных сервисов. Например, платные системы дистанционного обучения.
